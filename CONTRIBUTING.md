# Contributing Guide

First off, thank you for considering contributing to the Package Integrity Classification pipeline! We welcome contributions that help push the boundary of sim-to-real machine learning.

## How to Contribute

### 1. Reporting Bugs
*   Check if the issue has already been reported.
*   Provide a clear, descriptive title.
*   Describe the exact steps to reproduce the issue, including your OS, Python version, and Isaac Sim version.

### 2. Suggesting Enhancements
*   Open an issue detailing the enhancement.
*   Explain the *why*—what problem does this solve in a sim-to-real pipeline?

### 3. Submitting Pull Requests
*   **Branching:** Create a new branch from `main` (e.g., `feature/add-new-augmentation`).
*   **Documentation:** Update the `README.md` or `WALKTHROUGH.md` if your changes impact the user/developer workflow.
*   **Code Quality:** Ensure your code is thoroughly documented with Google-style docstrings and utilizes explicit type hints.

### Setting up the Environment
Please review `WALKTHROUGH.md` for a comprehensive setup guide.

*By submitting a PR, you agree that your contributions will be licensed under the project's Apache 2.0 License.*
